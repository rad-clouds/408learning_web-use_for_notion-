💡 什么是红黑树？

红黑树是一种自平衡二叉查找树，通过对节点着色（红/黑）并用旋转来维持近似平衡。最大高度不超过 2×log₂(n+1)。

Java 的 TreeMap、C++ 的 std::map、Linux 内核的 CFS 调度器都基于红黑树。

📏 五条性质（定义）

  1. 每个节点要么是红色🔴，要么是黑色⚫
  2. 根节点必为黑色
  3. 每个叶节点 (NIL) 是黑色
  4. 红色节点的两个子节点必须都是黑色（即不存在两个连续红色）
  5. 从任一节点到其每个叶子的路径上，黑色节点数量相同（黑高相同）

🔄 插入修复的三种情况

插入新节点时默认为红色，然后检查并修复违反性质的情况：

情况叔节点颜色当前节点位置修复操作
Case 1 红色 🔴 任意 父、叔变黑，祖父变红，向上递归
Case 2 黑色 ⚫ LR 或 RL（折线） 旋转父节点，转为 Case 3
Case 3 黑色 ⚫ LL 或 RR（直线） 旋转祖父节点 + 变色，完成
🔑 记忆口诀：「叔红就变色，叔黑看形状；折线转直线，直线旋转完」

📊 复杂度分析

操作时间复杂度说明
查找O(log n)BST 搜索 + 树高 ≤ 2log₂(n+1)
插入O(log n)最多 2 次旋转 + O(log n) 次变色
删除O(log n)最多 3 次旋转
与 AVL 对比红黑树插入删除只需 O(1) 次旋转，AVL 需要 O(logn) 次 → 红黑树更适合写多读少的场景

💻 插入修复代码（核心）

void RB_Insert_Fixup(T, z) {
    while (z.parent.color == RED) {
        if (z.parent == z.grandparent.left) {
            y = z.grandparent.right;        // 叔节点
            if (y.color == RED) {           // ★ Case 1: 叔红
                z.parent.color = BLACK;
                y.color = BLACK;
                z.grandparent.color = RED;
                z = z.grandparent;          // 向上递归
            } else {
                if (z == z.parent.right) {  // ★ Case 2: LR
                    z = z.parent;
                    Left_Rotate(T, z);      // 转为 LL
                }
                // ★ Case 3: LL
                z.parent.color = BLACK;
                z.grandparent.color = RED;
                Right_Rotate(T, z.grandparent);
            }
        } else { /* 对称处理 */ }
    }
    T.root.color = BLACK;                   // 根必黑
}
📝 408 常考：
  • n 个节点的红黑树高度 ≤ 2log₂(n+1)
  • 插入最多 2 次旋转，删除最多 3 次旋转
  • 黑高：从该节点到叶子的黑色节点数（不含自身）
  • 红黑树是「近似平衡」，AVL 是「严格平衡」
