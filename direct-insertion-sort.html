💡 算法思想

每次将一个待排序的记录按其关键字大小插入到前面已排好序的子序列中，直到全部记录插入完成。

类比：打扑克牌时理牌的过程——手里已经有一些排好序的牌，摸到一张新牌，从右往左找到它该放的位置插进去。

🔑 核心操作：查找插入位置，后移元素，插入新元素

📋 算法步骤

  1. 从第 2 个元素开始（认为第 1 个元素已排序）
  2. 取出当前元素 A[i]，暂存为 key
  3. 在 A[0..i-1] 中从后往前找插入位置
  4. 比 key 大的元素统统后移一位
  5. 将 key 插入空出的位置

📊 复杂度分析

指标值说明
时间复杂度（最好）O(n)初始有序，每次只比较1次
时间复杂度（平均）O(n²)平均比较 n²/4 次
时间复杂度（最坏）O(n²)初始逆序，比较 n²/2 次
空间复杂度O(1)只需要一个辅助变量 key
稳定性稳定 ✅相等元素不交换位置
适用场景数据量小（n≤50）或基本有序时效率极高

💻 代码实现

void InsertionSort(int A[], int n) {
    int i, j, key;
    for (i = 1; i < n; i++) {        // 从第2个元素开始
        key = A[i];                   // 取出待插入元素（哨兵）
        j = i - 1;                    // 从已排序部分的末尾开始比较
        while (j >= 0 && A[j] > key) { // 找插入位置
            A[j + 1] = A[j];          // 比 key 大的元素后移
            j--;
        }
        A[j + 1] = key;               // 插入到正确位置
    }
}
📝 408 常考：直接插入排序的比较次数 = 移动次数 + n-1。当初始基本有序时，直接插入排序是所有简单排序中最快的。这也是希尔排序选择插入排序作为基础的原因。
