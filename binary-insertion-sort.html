💡 算法思想

在直接插入排序的基础上，用二分查找代替顺序查找来定位插入位置。
因为前半部分已经有序，所以可以用二分查找在 O(logn) 时间内找到插入位置。

🔑 关键区别：直接插入排序 → 顺序查找 O(n²)比较；折半插入排序 → 二分查找 O(nlogn)比较，但移动次数不变，仍为O(n²)

📋 算法步骤

  1. 从第 2 个元素开始
  2. 对 A[0..i-1] 执行二分查找，找到 A[i] 的插入位置 pos
  3. 将 A[pos..i-1] 全部后移一位
  4. 将 A[i] 放入 A[pos]
⚠️ 注意：二分查找时，当 A[mid] == key，为了保证稳定性，应继续在右半部分查找（让相等元素保持原顺序）。即：if (A[mid] ≤ key) low = mid + 1

💻 代码实现

void BinaryInsertionSort(int A[], int n) {
    int i, j, low, high, mid, key;
    for (i = 1; i < n; i++) {
        key = A[i];
        low = 0; high = i - 1;

        // ★ 二分查找插入位置
        while (low <= high) {
            mid = (low + high) / 2;
            if (A[mid] <= key)   // 为保证稳定性用 <=
                low = mid + 1;   // 在右半部分继续找
            else
                high = mid - 1;  // 在左半部分继续找
        }
        // 此时 low 就是插入位置

        // 元素后移
        for (j = i - 1; j >= low; j--)
            A[j + 1] = A[j];

        // 插入
        A[low] = key;
    }
}
📝 408 常考：折半插入排序只减少了比较次数，移动次数不变。因此总时间复杂度仍为 O(n²)。但当比较代价远大于移动代价时（如记录很大），折半插入排序优势明显。
